# Daily Learning — Agent Context

## What this is
Single-container app: React SPA + FastAPI backend + SQLite. Generates one system design topic daily via GROQ LLM, delivers via browser push + email. Deployed on Cloud Run.

---

## Deployment

**Live URL**: https://daily-learning-taxnvq53va-uc.a.run.app
**GCP project**: `my-stuff-ai`, region `us-central1`
**Service name**: `daily-learning`

**How to redeploy**: `gcloud builds submit --config cloudbuild.yaml --project=my-stuff-ai --substitutions="_GROQ_API_KEY=...,..."`. No Artifact Registry setup needed — `cloudbuild.yaml` uses `gcloud run deploy --source` which lets Cloud Build handle image storage automatically in a managed `cloud-run-source-deploy` repo.

## Architecture decisions worth knowing

**Single process, no queue**: APScheduler runs inside the FastAPI process (not Celery). Fires at 15:30 UTC (= 9PM IST). Cloud Run must have `--min-instances=1` or the scheduler dies on scale-to-zero. `/api/admin/trigger` exists as a Cloud Scheduler HTTP fallback.

**Static files served by FastAPI**: Multi-stage Dockerfile builds React → copies `frontend/dist/` into the Python image. FastAPI mounts `/assets` as StaticFiles and catches everything else with a SPA fallback route returning `index.html`.

**SQLite with named volume**: `DATABASE_URL=sqlite:////app/data/topics.db`. Docker volume `sysdesign-data` persists across rebuilds. Data is lost on `docker compose down -v`. No ORM migrations — `Base.metadata.create_all` on startup.

**Admin auth**: All `/api/admin/*` routes check `X-Admin-Key` header against `ADMIN_KEY` env var. If `ADMIN_KEY` is empty, auth is skipped (dev convenience).

## LLM / structured output

Model: `gpt-oss-120b` via GROQ. Uses **strict structured output** (`response_format.type = "json_schema"`, `strict: True`) — not the older `json_object` mode. Schema is `TOPIC_SCHEMA` in `backend/llm.py`. Key constraints:
- `domain` is an enum (14 values) — cannot hallucinate new domains
- `difficulty` is an enum (`Beginner | Intermediate | Advanced`)
- `additionalProperties: false` required by GROQ strict mode
- All previous topic **titles** (not IDs) are injected into the prompt to prevent duplicates
- No JSON parsing guards needed — strict mode guarantees compliance

## Data model

`Topic`: `date` (unique, YYYY-MM-DD), `title`, `domain`, `difficulty`, `problem_statement`, `context_text`, `deep_dive`, `real_world_examples`, `solution_approaches`, `key_takeaways` (JSON array), `further_reading` (JSON array).

One topic per day enforced at DB level via `unique=True` on `date`. `trigger` skips if today exists; `trigger-force` deletes then regenerates.

`PushSubscription`: `endpoint` (unique), `p256dh`, `auth`. Stale subscriptions (HTTP 404/410 from push provider) are auto-deleted after a failed send in `admin.py:run_daily_generation`.

## Frontend patterns

- **Only `main-content` scrolls.** `html, body { overflow: hidden }`, `app-layout` is `height: 100vh`. Sidebar is static — do not add `overflow` or scrolling to `.sidebar` or `.app-body`.
- **Sidebar collapse**: desktop = `width` transition (0 ↔ 260px). Mobile = `position: fixed` + `transform: translateX`. Never mix these — the `!important` overrides in the mobile breakpoint are intentional.
- **Sidebar defaults open** (`useState(true)` in `App.jsx`).
- **Deep links**: `?topic={id}` param, set via `pushState` on select.
- **Poll interval**: topics list refreshes every 5 minutes via `setInterval` in `App.jsx`.
- **Notification dismiss**: `localStorage.getItem('notifyDismissed') === '1'` — banner never shows again once dismissed.

## Design system

**Theming**: CSS variables on `:root` (dark) and `[data-theme="light"]` on `<html>`. Toggled via `document.documentElement.setAttribute`. Persisted in `localStorage`.

**Typography**:
- Body/UI: system-sans (`-apple-system, BlinkMacSystemFont, 'Segoe UI'`)
- Topic titles (`h1.topic-title`): Georgia serif
- App logo: `var(--font-mono)` with a typewriter animation on mount and a blinking `▊` cursor after completion (see `TypewriterLogo` in `Header.jsx`). Logo text is `daily_learning` (underscore, lowercase).
- Dates/mono: `font-family: var(--font-mono)`

**Section headers**: accent blue (`var(--accent)`), 10px uppercase, `letter-spacing: 0.14em`, with a `::before` pseudo-element (16×2px bar). Sections separated by `.section + .section { border-top: 1px solid var(--border-subtle) }` — no manual divider elements.

**Icons**: Feather set from `react-icons/fi` exclusively. Domain icons in sidebar map domain strings to Fi* components in `Sidebar.jsx`. Header uses `FiSidebar`, `FiBell`/`FiBellOff`, `FiSun`/`FiMoon`.

**Emoji policy**: minimal — no emoji in section labels, title, or prose. Domain icons in the sidebar use react-icons, not emoji.

## Email

Subject: `[Important] Daily Learning — {title}`. Sent via `aiosmtplib` with `start_tls=True`. Footer reads `Daily Learning · Generated by gpt-oss-120b`. No emoji in email template.

## Push notifications

VAPID keys generated once via `GET /api/admin/generate-vapid`, stored in env vars. Same key pair works across all environments (local + Cloud Run). Service worker at `frontend/public/sw.js` — handles push events and notification clicks.
